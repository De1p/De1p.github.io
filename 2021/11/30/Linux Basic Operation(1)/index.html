Basic

在linux中，大多数命令都有帮助文档，当您不知道这个命令怎么使用，亦或是您忘了某个参数时，您可以尝试–help参数以查看帮助文档

如果您遇到无法执行指令时，请先查看您当前是否处于非root用户下，您可以查看命令指示符是否为#，亦或者您可以使用whoami命令查看当前的用户名

如果您处于非root用户下时，您可以使用su命令来进行切换root用户

内容

一.基本操作语句介绍

操作路径指令

Linux下目录基础知识

语法 描述
~ 用户目录
/ 根目录
./ 当前目录
./ 上一级目录
* 表所有

移动路径指令(cd)

查看路径下文件指令(ls)

参数 描述
-l 列出目录下子目录和文件的详细信息
-a 列出目录下子目录和文件的详细信息
-t 按最后修改时间排序
-S 按文件大小排序（大写的S）
-r 排序时按倒序

创建文件指令(touch)

参数 描述
-a 只修改文件的access(访问)时间
-c 或–no-create 不创建不存在的文件
-d 使用指定的日期时间，而非现在的时间
-m 指修改Modify(修改)时间,而不修改access(访问)时间
-r file 使用指定file文件的时间戳（access，modify）更新文件的时间戳（access，modify）
-t 将时间修改为参数指定的日期，如：07081556代表7月8号15点56分

创建文件夹指令(mkdir)

参数 描述
-m 模式，设定权限<模式> (类似 chmod)，而不是 rwxrwxrwx 减 umask
-p 可以是一个路径名称。此时若路径中的某些目录尚不存在,加上此选项后,系统将自动建立好那些尚不存在的目录,即一次可以建立多个目录;
-v 每次创建新目录都显示信息

文件操作指令

关于路径的基础知识

绝对路径

绝对路径是从linux最底层的根(/)开始算的。

相对路径

相对路径是直接输入当前路径下的文件夹或者以 . 或 .. 开始算的，.表示用户当前操作所处的位置，.. 表示上一级目录。

举个例子

/home/oracle 这个路径就是绝对路径
./home/oracle 这个就是相对路径，例如现在在/test路径下，此条路径换算成绝对路径为/test/home/oracle
../home/oracel 这个也是相对路径，例如现在在/test路径下，此条路径换算成绝对路径为：/home/oracel
注意后面两个的对比，..是回到上一级然后在往后算。

您可以使用pwd命令查看当前所在的绝对路径

移动文件指令(mv)

参数 描述
-b 若需覆盖文件，则覆盖前先行备份
-f 强制的意思，如果目标文件已经存在，不会询问而直接覆盖
-i 若目标文件 (destination) 已经存在时，就会询问是否覆盖
-u 若目标文件已经存在，且 source 比较新，才会更新(update)
-t 指定mv的目标目录，该选项适用于移动多个源文件到一个目录的情况，此时目标目录在前，源文件在后

执行操作时可能会遇到需要移动文件夹中的部分相同后缀文件，此时可以使用*

例如想移动文件夹中以txt为后缀的所有文件

复制文件指令(cp)

参数 描述
-a 此选项通常在复制目录时使用，它保留链接、文件属性，并复制目录下的所有内容
-d 复制时保留链接。这里所说的链接相当于 Windows 系统中的快捷方式
-f 覆盖已经存在的目标文件而不给出提示
-i 与 -f 选项相反，在覆盖目标文件之前给出提示，要求用户确认是否覆盖，回答 y 时目标文件将被覆盖
-p 除复制文件的内容外，还把修改时间和访问权限也复制到新文件中
-r 若给出的源文件是一个目录文件，此时将复制该目录下所有的子目录和文件
-l 不复制文件，只是生成链接文件

课后实训

实训任务：对linux进行基本的文件操作，对相关目录执行添加、删除、修改、创建。

实验环境：kali虚拟机

实训1：移动，创建目录

  1. 在用户目录下创建两个文件夹名为test1、test2

  2. 进入test1文件夹创建一个文件名为hzvtc

  3. 进入test2文件夹创建一个文件名为名字缩写+学号

  4. 在hzvtc文件中添加名字缩写加学号

  5. 查看hzvtc文件

  6. 删除test2文件夹中的文件

  7. 查看用户目录底下的文件夹和文件

  8. 将hzvtc移动到test2中

  9. 将test2的hzvtc复制到test1中

拓展任务：

  1. 查看test1中文件的权限

  2. 将test1中文件权限改为文件所有者可读写运行、群组可读写、匿名用户可读

  3. 直接删除test1文件夹

  4. 尝试用一条指令同时创建两个文件夹

  5. 尝试用一条指令同时执行两个操作 且其中一个不执行另一个执行失败
