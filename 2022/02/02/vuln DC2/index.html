Vulnhub DC2靶机渗透过程

发表于2022-02-02|更新于2022-02-02|📖乌云靶机
|阅读量:

前言

本篇文章为自我学习的wooyun平台靶机，主要为自己对靶机的理解渗透，和在渗透学习过程中遇到的问题以及相对应的解决办法

Basic

在阅读本篇文章前，您需要掌握如下（包括且不止于）msfconsole，fping，nmap，sql注入…等

在复现本篇文章前，您需要准备，并保证其属于同一网络下：

靶机下载地址：https://download.vulnhub.com/dc/DC-2.zip

内容.

0x00靶机要求与描述：

DC-2 是一个专门构建的易受攻击的实验室，旨在获得渗透测试领域的经验。共有五个flag，但最终目标是拿到root目录中的flag。

0x01信息收集与ip确定：

1.确定ip

因为我们不知道dc2靶机的ip地址所以我们将其放在与kali同一个局域网中

1.探测目标ip

方法一：使用fping工具

方法二：使用arp-scan工具

方法三：使用 netdiscover工具

2.信息收集

使用nmap或zenmap对目标ip进行扫描

0x02信息分析及漏洞利用

在信息收集后我们得知靶机80端口开放，同时这是一个基于wordpress搭建的网页

1.信息收集

在进入网站后我们发现其中包含着一个flag的分栏

此处提示我们使用cewl生成一个通用的字典，并且告诉了我们flag的位置

先使用目录扫描工具对网页进行一个简单的扫描

同时在扫描的过程中我们发现了很多个目录遍历漏洞，由于文件数量太多，且没有在名字上看到敏感信息的文件

接下来使用wpscan工具尝试枚举账号密码

得到了三个用户名将其保存到字典中，接下来根据提示使用cewl工具爬取网页关键字生成爆破字典，并使用wpscan工具进行爆破

得到了两个账户，登录后台尝试收集信息

在文章页发现了flag2

在尝试了多种方法未果后，尝试使用nmap扫描工具对全端口进行进一步扫描尝试获得更多信息

在扫描完成后发现其7744端口存在ssh服务

尝试用之前的字典对ssh进行爆破

得到ssh的账户和密码login: tom password: parturient

随后进行链接ssh，发现是rbash，尝试对rbash进行提权

提权过程

先检查可用命令，发现sudo su / ftp cp等命令都被禁止了 但是vi可以使用

尝试用vi的特性在vi中也可以执行命令

同时在翻阅资料的过程中还找到另一种逃逸rbash的方式

在得到完整shell之后根据提示切换到jerry账户，但我们在ssh爆破中发现用原先的密码登录不进去，有两种可能，可能是jerry的系统密码和网页密码不一致，也可能是jerry不支持远程ssh登录，所以还是用原先的密码尝试，之后发现可以登录

在flag4中作者和我们玩了个小小的文字游戏，但是还是能看出提示我们使用git来进行提权

sudo -l 我们可以发现无需root权限，jerry 可以使用 git

利用git的help可以执行命令的特性（且具有uid权限）

至此得到了root的权限，根据作者的尿性，果然在root的用户目录底下找到了最终的flag
